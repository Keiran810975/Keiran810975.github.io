浅拷贝（Shallow Copy） 和 深拷贝（Deep Copy） 是两种不同的对象复制方式，用于在创建一个新对象时决定如何处理原对象的成员变量，特别是引用类型的变量。
浅拷贝:
浅拷贝会复制对象的所有字段，但如果字段是引用类型（如数组、对象），浅拷贝只复制引用而不复制实际的对象。
结果是新对象和原对象共享相同的引用对象。如果其中一个对象修改了共享的引用对象，另一个对象也会受到影响。
在 Java 中，可以通过实现 Cloneable 接口并重写 Object 类的 clone() 方法实现浅拷贝。
例如，数组、对象等的引用在浅拷贝中是共享的。
深拷贝:
深拷贝会递归地复制对象的所有字段，包括所有引用类型的对象，确保新对象和原对象完全独立。
深拷贝的结果是，新对象拥有与原对象相同的值，但完全独立于原对象，不共享任何引用。
实现深拷贝通常需要自定义拷贝逻辑，可以通过实现 clone() 方法并在其中深度克隆每一个引用对象，或者通过序列化（将对象序列化然后再反序列化）来实现。

(2)
要为一个类实现深拷贝并避免浅拷贝问题，可以采取以下方法：
递归拷贝:
对于类中的每一个引用类型的字段，递归地调用它们的 copy() 或 clone() 方法，以创建完全独立的副本。
例如，如果类中包含一个数组字段，则需要手动复制每个元素。
构造函数拷贝:
创建一个新的构造函数，接受一个对象并复制其所有字段，包括递归地复制所有引用类型的字段。

(3)
== 运算符:
== 比较的是两个引用变量是否指向同一个对象或内存地址。
对于 String 来说，== 比较的是两个字符串变量的引用是否相同。
equals() 方法:
equals() 方法是 Object 类中定义的，用于比较两个对象的内容是否相等。对于 String 类，equals() 方法被重写用于比较字符串的内容是否相同。

Java 的可变参数允许一个方法接受多个参数，而无需显式定义每个参数。可变参数用 … 表示，在方法内部，Java 将可变参数处理为一个数组。可变参数必须放在参数列表的最后一个。
实现机制：
在编译时，Java 将可变参数转换为一个数组。
方法可以通过数组的方式访问这些参数。
这样的方法调用可以接受零个或多个参数。

(2)
能
strscat(new String[]{“a”, “b”}) 能通过编译。因为可变参数在编译时会被转换为数组形式，所以直接传递一个 String[] 数组是可以的。这与 strscat(String… args) 接受数组的逻辑是一致的。

(3)
如果存在 String strscat(String[] args)，代码仍然可以通过编译，但会有编译警告或报错，因为这种情况导致方法签名冲突

(4)
不能通过编译。Java 不会自动将单个的多个参数转换为数组传递，因此调用 strscat(“a”, “b”, “c”) 会报错。
