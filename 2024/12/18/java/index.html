草

封装

成员变量/方法

  • private: 只有当前类可以访问，其他类不能直接访问
  • protected：当前类、子类以及同包中的其他类可以访问,但外部类无法访问.如果当前类与目标类不在同一个包中，并且当前类不是目标类的子类，那么protected成员是不可访问的。
  • default：如果没有指定访问控制符，则默认为default访问级别，表示只有同一包内的类可以访问。
  • static: 它修饰的方法或变量属于类而不是类的实例。使用static修饰的成员被类的所有实例共享。静态方法里,无this
  • final: 不可改变,final 方法不能被子类重写（Override）,final类不能被继承。
  • 静态方法不能访问普通成员变量，因为静态方法属于类，而普通成员变量属于对象，它们的生命周期和访问方式不同。
  • 静态方法中只能直接调用静态成员或者方法，不能调用非静态方法或者非静态成员，

局部变量和成员变量区别

  • 成员变量存储在堆内存的对象中
  • 局部变量存储在栈内存的方法中
  • 成员变量随着对象的出现而出现在堆中，随着对象的消失而从堆中消失
  • 局部变量随着方法的运行而出现在栈中，随着方法的弹栈而消失
  • 成员变量因为在堆内存中，所有成员变量具有默认的初始化值
  • 局部变量没有默认的初始化值，必须手动的给其赋值才可以使用。

重载

指一个类中可以定义有相同的名字，但参数不同的多个方法，调用时会根据不同的参数表选择对应的方法
即使参数的类型相同，但顺序不同也能实现重载。

重写

toString()方法是Object类中的一个方法，所有 Java 类都默认继承了这个方法

构造方法

不带参数构造方法（默认构造方法）
New关键字的作用{
为对象分配内存空间。
引起对象构造方法的调用。
为对象返回一个引用。
}

无参构造函数创建对象时，成员变量的值被赋予了数据类型的隐含初值

在Java里，如果一个类没有明显的表明哪一个类是它的父类，Object类就是它的父类
如果程序员为类定义了构造函数，Java就不会为该类创建默认的不带参数的构造函数。
如果类中提供的构造函数都不是无参数构造函数，却企图调用无参数构造函数初始化此类的对象，编译时会产生语法错误

类之间的关系

  • 关联：类A与类B的实例之间存在特定的对应关系。
  • 依赖：类A访问类B提供的服务。
  • 聚集：类A为整体类，类B为局部类，类A的对象由类B的对象组合而成。
  • 泛化：类A继承类B。
  • 实现：类A实现了B接口。

类是引用类型变量

垃圾自动回收机制

  • Java虚拟机后台线程负责内存的回收
  • 垃圾强制回收机制：Java系统提供了方法System.gc()和Runtime.gc() 方法来强制立即回收垃圾(但系统并不保证会立即进行垃圾回收)
  • 判断一个存储单元是否是垃圾的依据是：该存储单元所对应的对象是否仍被程序所用。
  • 判断一个对象是否仍为程序所用的依据是：是否有引用指向该对象。
  • 当确定对象不再需要使用时，可以通过将引用变量赋值为 null 来清除对对象的引用。

匿名对象

  • 没有引用变量：匿名对象创建后不能通过变量再次访问。
  • 生命周期短：只能在创建它的语句中使用，随后失去引用，等待垃圾回收。
  • 常用于一次性操作：例如作为方法参数或返回值。

浅拷贝

浅拷贝是指创建一个新对象，但新对象的非基本类型字段只复制引用，而不是复制引用指向的实际内容。
这意味着浅拷贝后的新对象和原对象共享引用类型字段指向的同一个对象。

在 Java 中，浅拷贝通常通过实现 java.lang.Cloneable 接口，并重写 Object 类的 clone() 方法来实现。

深拷贝

深拷贝是指创建一个新对象，并且复制原对象中的所有内容，包括引用类型字段所指向的对象，也会递归地创建新的副本，而不是仅仅复制引用。
深拷贝使得新对象与原对象完全独立，即修改一个对象的内容不会影响另一个对象。

通过 Java 的序列化机制，将对象序列化为字节流再反序列化为新对象，直接实现深拷贝。

对象序列化

代码块

构造代码块会在创建对象时被调用，每次创建时都会被调用，优先于类构造函数执行
静态构造代码块只在类第一次加载时执行一次，之后不再执行，而非静态代码块在每new一次就执行一次

继承

草

  • 如果子类只能有一个直接父类，称为单继承; 如果子类可以有多个直接父类，称为多继承
  • Java不支持类的多继承
  • 子类不能直接访问父类的私有属性和方法，但是可以通过父类的 public 或protected方法访问
  • 子类无法继承父类的构造方法，但是可以通过调用父类的构造方法来初始化子类的属性。
  • 如果子类的构造函数没有显式调用 super()，编译器会在子类构造函数的第一行隐式地插入 super() 调用父类的无参构造函数
  • 构造函数中调用 this()（调用本类的其他构造函数）或 super()（调用父类的构造函数）必须是构造函数体中的第一条语句。如果同时使用 this() 和 super()，会导致语法错误，因为它们都要求出现在第一行。

向上转型

向上转型只能调用父类中定义的属性和方法，对于子类中的方法和属性它就不行了，必须强制转成子类类型

变量隐藏

在子类对父类的继承中，如果子类的成员变量和父类的成员变量同名，此时称为子类隐藏override了父类的成员变量
子类若要引用父类的同名变量。要用super关键字做前缀加圆点操作符引用,即super.变量名

私有方法、静态方法不能被覆盖，如果在子类出现了同签名的方法，就是方法隐藏

继承是一种强耦合关系。父类变，子类就必须变

多态

草

  • 静态多态性：包括方法的重载；
  • 动态多态性：在编译时不能确定所要调用的方法，只有在运行时才能确定所要调用的方法，又称为运行时的多态性。
    继承是子类使用父类的方法，而多态则是父类使用子类的方法。 (把父类当做子类来用)

多态只能调用父类中定义的方法和变量，即使子类重写了方法，仍然以父类声明为准

父类引用只能访问父类中定义的方法和属性，不能直接调用子类的特有方法或属性,可以类型强转

静态方法不支持多态:调用时由引用的类型决定，而非实际对象

抽象类

  • 如果一个类里有抽象的方法，则这个类就必须声明成抽象的。但一个抽象类中却可以没有抽象方法。
  • 抽象方法不能被private、final或static修饰
  • 抽象类是使用 abstract 关键字修饰的类，不能直接实例化

抽象类的引用：
抽象类的引用可以指向其具体子类的对象。这种方式支持多态，使得程序具有更好的扩展性和灵活性。

接口

设计模式

单例模式

一个类在内存中只有一个实例（对象）存在，该类一般没有属性。
无法继承，所以无法扩展，无法更改它的实现。

饿汉式

在类加载时立即创建实例，线程安全，适合单例对象占用内存较小或初始化开销较低的场景。

懒汉式

实例在第一次使用时创建，延迟加载，节省内存，但需要注意线程安全问题。

工厂模式

定义一个创建产品对象的工厂接口，将产品对象的实际创建工作推迟到具体子工厂类当中。
这满足创建型模式中所要求的“创建与使用相分离”的特点。

简单工厂模式

也称为静态工厂方法模式，定义了一个工厂类，通过判断传入的参数来决定实例化哪一个具体的产品类

工厂方法模式

一种更具扩展性的工厂模式，它将产品的创建交给子类。客户端只需要依赖于抽象工厂类，而无需关心具体的产品类是如何创建的。

抽象工厂模式

提供一个接口，用于创建一系列相关或相互依赖的对象，而无需指定它们的具体类。抽象工厂模式通常与多个产品族（不同类别的产品）一起使用。
