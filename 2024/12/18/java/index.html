Java期末复习

Posted on 2024-12-18 Edited on 2024-12-26

草

封装

成员变量/方法

  • private: 只有当前类可以访问，其他类不能直接访问
  • protected：当前类、子类以及同包中的其他类可以访问,但外部类无法访问.如果当前类与目标类不在同一个包中，并且当前类不是目标类的子类，那么protected成员是不可访问的。
  • default：如果没有指定访问控制符，则默认为default访问级别，表示只有同一包内的类可以访问。
  • static: 它修饰的方法或变量属于类而不是类的实例。使用static修饰的成员被类的所有实例共享。静态方法里,无this
  • final: 不可改变,final 方法不能被子类重写（Override）,final类不能被继承。
  • 静态方法不能访问普通成员变量，因为静态方法属于类，而普通成员变量属于对象，它们的生命周期和访问方式不同。
  • 静态方法中只能直接调用静态成员或者方法，不能调用非静态方法或者非静态成员，

局部变量和成员变量区别

  • 成员变量存储在堆内存的对象中
  • 局部变量存储在栈内存的方法中
  • 成员变量随着对象的出现而出现在堆中，随着对象的消失而从堆中消失
  • 局部变量随着方法的运行而出现在栈中，随着方法的弹栈而消失
  • 成员变量因为在堆内存中，所有成员变量具有默认的初始化值
  • 局部变量没有默认的初始化值，必须手动的给其赋值才可以使用。

重载

指一个类中可以定义有相同的名字，但参数不同的多个方法，调用时会根据不同的参数表选择对应的方法
即使参数的类型相同，但顺序不同也能实现重载。

重写

toString()方法是Object类中的一个方法，所有 Java 类都默认继承了这个方法

构造方法

不带参数构造方法（默认构造方法）
New关键字的作用{
为对象分配内存空间。
引起对象构造方法的调用。
为对象返回一个引用。
}

无参构造函数创建对象时，成员变量的值被赋予了数据类型的隐含初值

在Java里，如果一个类没有明显的表明哪一个类是它的父类，Object类就是它的父类
如果程序员为类定义了构造函数，Java就不会为该类创建默认的不带参数的构造函数。
如果类中提供的构造函数都不是无参数构造函数，却企图调用无参数构造函数初始化此类的对象，编译时会产生语法错误

类之间的关系

  • 关联：类A与类B的实例之间存在特定的对应关系。
  • 依赖：类A访问类B提供的服务。
  • 聚集：类A为整体类，类B为局部类，类A的对象由类B的对象组合而成。
  • 泛化：类A继承类B。
  • 实现：类A实现了B接口。

类是引用类型变量

垃圾自动回收机制

  • Java虚拟机后台线程负责内存的回收
  • 垃圾强制回收机制：Java系统提供了方法System.gc()和Runtime.gc() 方法来强制立即回收垃圾(但系统并不保证会立即进行垃圾回收)
  • 判断一个存储单元是否是垃圾的依据是：该存储单元所对应的对象是否仍被程序所用。
  • 判断一个对象是否仍为程序所用的依据是：是否有引用指向该对象。
  • 当确定对象不再需要使用时，可以通过将引用变量赋值为 null 来清除对对象的引用。

匿名对象

  • 没有引用变量：匿名对象创建后不能通过变量再次访问。
  • 生命周期短：只能在创建它的语句中使用，随后失去引用，等待垃圾回收。
  • 常用于一次性操作：例如作为方法参数或返回值。

浅拷贝

浅拷贝是指创建一个新对象，但新对象的非基本类型字段只复制引用，而不是复制引用指向的实际内容。
这意味着浅拷贝后的新对象和原对象共享引用类型字段指向的同一个对象。

在 Java 中，浅拷贝通常通过实现 java.lang.Cloneable 接口，并重写 Object 类的 clone() 方法来实现。

深拷贝

深拷贝是指创建一个新对象，并且复制原对象中的所有内容，包括引用类型字段所指向的对象，也会递归地创建新的副本，而不是仅仅复制引用。
深拷贝使得新对象与原对象完全独立，即修改一个对象的内容不会影响另一个对象。

通过 Java 的序列化机制，将对象序列化为字节流再反序列化为新对象，直接实现深拷贝。

对象序列化

代码块

构造代码块会在创建对象时被调用，每次创建时都会被调用，优先于类构造函数执行
静态构造代码块只在类第一次加载时执行一次，之后不再执行，而非静态代码块在每new一次就执行一次

继承

草

  • 如果子类只能有一个直接父类，称为单继承; 如果子类可以有多个直接父类，称为多继承
  • Java不支持类的多继承
  • 子类不能直接访问父类的私有属性和方法，但是可以通过父类的 public 或protected方法访问
  • 子类无法继承父类的构造方法，但是可以通过调用父类的构造方法来初始化子类的属性。
  • 如果子类的构造函数没有显式调用 super()，编译器会在子类构造函数的第一行隐式地插入 super() 调用父类的无参构造函数
  • 构造函数中调用 this()（调用本类的其他构造函数）或 super()（调用父类的构造函数）必须是构造函数体中的第一条语句。如果同时使用 this() 和 super()，会导致语法错误，因为它们都要求出现在第一行。

向上转型

向上转型只能调用父类中定义的属性和方法，对于子类中的方法和属性它就不行了，必须强制转成子类类型

变量隐藏

在子类对父类的继承中，如果子类的成员变量和父类的成员变量同名，此时称为子类隐藏override了父类的成员变量
子类若要引用父类的同名变量。要用super关键字做前缀加圆点操作符引用,即super.变量名

私有方法、静态方法不能被覆盖，如果在子类出现了同签名的方法，就是方法隐藏

继承是一种强耦合关系。父类变，子类就必须变

泛型

允许在类、接口和方法中使用类型参数的机制，可以用来编写具有强类型检查能力且更通用的代码。通过泛型，可以在编译时检查类型安全，并避免类型转换错误。

通配符

? 通配符：表示未知类型，可以用在类、方法或接口中。

上界通配符 (<? extends T>)：表示类型参数必须是 T 或其子类。

下界通配符 (<? super T>):表示类型参数必须是 T 或其父类。

类型擦除

Java 的泛型在编译后会进行类型擦除，泛型类型会被替换为原始类型（如 Object）。这就是为什么泛型不能用于基本数据类型（如 int、double）。

注意事项

  • 泛型不能用于静态方法或静态变量：静态成员属于类，不属于实例，而泛型参数属于实例。
  • 不能直接创建泛型数组：T[] array = new T[10]; 是不允许的，因为类型擦除会导致数组运行时的类型信息丢失。

多态

Java实现多态有三个必要条件：继承、方法覆盖、向上转型。

草

  • 静态多态性：包括方法的重载；
  • 动态多态性：在编译时不能确定所要调用的方法，只有在运行时才能确定所要调用的方法，又称为运行时的多态性。
  • 继承是子类使用父类的方法，而多态则是父类使用子类的方法。 (把父类当做子类来用)
  • 向上转型：在多态中需要将子类的对象赋给父类引用，只有这样该引用才能够具备调用父类的方法和子类的方法的能力。

多态只能调用父类中定义的方法和变量，即使子类重写了方法，仍然以父类声明为准

父类引用只能访问父类中定义的方法和属性，不能直接调用子类的特有方法或属性,可以类型强转

静态方法不支持多态:调用时由引用的类型决定，而非实际对象

抽象类

  • 如果一个类里有抽象的方法，则这个类就必须声明成抽象的。但一个抽象类中却可以没有抽象方法。
  • 抽象方法不能被private、final或static修饰
  • 抽象类是使用 abstract 关键字修饰的类，不能直接实例化

抽象类的引用：
抽象类的引用可以指向其具体子类的对象。这种方式支持多态，使得程序具有更好的扩展性和灵活性。

接口

  • 可以包含抽象方法（没有方法体），也可以包含常量（静态常量）
  • 一个类可以实现多个接口
  • 接口中的方法通常不包含实现，只有方法的签名（即方法名、参数列表和返回类型），但接口方法默认是 public abstract 的（可以省略修饰符）
  • 常量：接口可以包含常量，常量是 public static final 的（也可以省略修饰符）
  • 接口不能被实例化，因此不能创建接口的对象。只能通过类实现接口来间接使用接口。

一个类实现多个接口

接口可以包含默认方法（default 方法），这些方法有方法体，允许接口提供默认的实现。实现接口的类可以选择性地覆盖这些默认方法。
还允许接口定义静态方法。这些静态方法不能被实现类覆盖，必须通过接口本身来调用。

接口回调

接口和抽象类的共同点/区别

共同点

  • 都不能被实例化
  • 在语义上，都位于系统的抽象层，需要其他类来进一步提供实现细节。
  • 抽象类与接口都是为了继承与多态，它们都需要子类来继承或实现才有意义，最终目的是为了多态；
  • 子类重写了父类的方法，再通过向上转型，由父类对象引用指向子类对象，达到运行时动态调用子类方法的目的。

区别

  • 接口中的成员变量和方法只能是public类型的，而抽象类中的成员变量和方法可以处于各种访问级别。
  • 接口中的成员变量只能是public、static和final类型的，而在抽象类中可以定义各种类型的实例变量和静态变量。
  • 接口中没有构造方法，抽象类中有构造方法。接口中所有方法都是抽象方法，抽象类中可以有，也可以没有抽象方法。抽象类比接口包含
  • 抽象类是某一类事物的一种抽象，而接口不是类，它只定义了某些行为
  • 在语义上，接口表示更高层次的抽象，声明系统对外提供的服务。而抽象类则是各种具体类型的抽象。

对象的向下映射就不一定了，有时可以，有时不可以，如果不可以转的话，程序是不会报语法错误的，发生的是一个运行时异常。

设计原则

  • 低耦合：是指软件系统中，模块与模块之间的直接依赖程度低。
  • 高内聚：系统存在AB两个模块儿进行交互，如果修改了A模块儿不影响B模块的工作，那么认为A模块儿有足够的内聚。

  • 开放封闭原则（Open-Closed Principle，OCP）：一个软件实体应当对扩展开放，对修改关闭。即软件实体应尽量在不修改原有代码的情况下进行扩展。

设计模式

创建型，结构型，行为型

创建型(涉及对象的实例化，这类模式的特点是，不让用户代码依赖于对象的创建或排列方式，避免用户直接使用new运算符创建对象。)

单例模式

一个类在内存中只有一个实例（对象）存在，该类一般没有属性。
无法继承，所以无法扩展，无法更改它的实现。

饿汉式

在类加载时立即创建实例，线程安全，适合单例对象占用内存较小或初始化开销较低的场景。

懒汉式

实例在第一次使用时创建，延迟加载，节省内存，但需要注意线程安全问题。

工厂模式

定义一个创建产品对象的工厂接口，将产品对象的实际创建工作推迟到具体子工厂类当中。
这满足创建型模式中所要求的“创建与使用相分离”的特点。

简单工厂模式

也称为静态工厂方法模式，定义了一个工厂类，通过判断传入的参数来决定实例化哪一个具体的产品类

工厂方法模式

一种更具扩展性的工厂模式，它将产品的创建交给子类。客户端只需要依赖于抽象工厂类，而无需关心具体的产品类是如何创建的。

抽象工厂模式

提供一个接口，用于创建一系列相关或相互依赖的对象，而无需指定它们的具体类。抽象工厂模式通常与多个产品族（不同类别的产品）一起使用。

原型模式

通过复制现有对象来创建新的对象，而不是通过实例化类。它使用一个原型接口来定义具体类的克隆方法
当创建对象成本较高时，可以通过复制减少开销。

深拷贝实现

建造者模式

将对象的构建过程与对象的表示分离，使得同样的构建过程可以创建不同的表示
当一个对象有复杂的内部结构，需要通过多个步骤才能创建完成。
当构建过程需要灵活、可配置，并可能创建多种表示形式。

结构型

适配器模式

将一个类的接口转换成客户期望的另一个接口，使得原本由于接口不兼容而不能一起工作的类能够协同工作

类适配器模式(通过继承实现适配器模式。)
对象适配器模式(通过组合实现适配器模式。)

系统中有功能良好的类，但它的接口与现有系统不兼容，可以通过适配器进行复用
使用第三方库时，库的接口与系统接口不一致，可以通过适配器来桥接

装饰模式

在不修改现有类的基础上动态地给对象添加新的职责（功能）。
装饰器模式的核心思想： “增强对象功能的灵活性，且不破坏原有类结构”。
例：假设我们有一个咖啡类，需要根据不同的配料动态计算价格，而不修改咖啡类本身。

被装饰者和装饰者是松耦合关系

外观模式

通过提供一个统一的接口来简化子系统的使用，使得子系统中的一组接口变得更容易访问。
外观模式的核心思想： “对外隐藏复杂性，提供一个简化的接口”。
假设我们有一个家庭影院系统，包含多个组件（灯光、音响、播放器等），外观模式可以将复杂的组件操作封装为一个简单的接口，方便用户一键启动或关闭。

增加新的子系统可能需要修改外观类或客户端的源代码，违背了“开闭原则
使用抽象外观类可以解决这个问题。

代理模式

通过一个代理对象控制对目标对象的访问。代理对象充当客户端和目标对象之间的中介，可以对目标对象的操作进行额外的控制。
为对象提供一个代理，以便在不改变目标对象的前提下控制访问、增加功能或者优化性能。

策略模式

定义了一组算法，将每个算法封装到独立的类中，并使它们可以互相替换。策略模式使得算法可以在不影响客户端的情况下发生变化
核心就是将类中经常需要变化的部分分割出来，并将每种可能的变化对应地交给抽象类的一个子类或实现接口的一个类去负责，
从而让类的设计者不去关心具体实现，避免所设计的类依赖于具体的实现

上下文和具体策略松耦合，策略模式满足开闭原则

访问者模式

定义了一种新的操作方式，通过将操作封装到一个对象（访问者）中，从而分离数据结构和操作逻辑
将对象结构和行为分离，通过访问者为对象结构中的元素定义新的操作

例：员工系统，访问者对员工进行绩效评估和薪资调整

需要对集合中的对象进行很多不同的并且不相关的操作，而我们又不想修改对象的类，就可以使用访问者模式。

责任链模式

责任链模式是使用多个对象处理用户请求的成熟模式,责任链模式的关键是将用户的请求分派给许多对象
例：用户请求的审批流程

观察者模式

定义一种一对多的依赖关系，使得一个对象（被观察者）的状态发生变化时，所有依赖它的对象（观察者）都会收到通知并自动更新
被观察者通知所有的观察者，保持状态一致
天气预报系统，观察者订阅天气变化

异常

异常是一个可以正确运行的程序在运行中可能发生的错误。
异常分为：受检查异常和非受检查异常。

  • 受检查异常：必须在编译时处理，否则编译不通过。比如IOException。必须用try-catch或throws声明。
  • 非受检查异常：不需要在编译时处理，也可以不用try-catch或throws声明。比如NullPointerException。

常见异常类

NullPointerException: 访问空引用时抛出。
ArithmeticException: 算术运算错误（如除以零）。
ArrayIndexOutOfBoundsException: 访问数组越界。
ClassCastException: 类型转换错误。
IOException: 输入输出错误。
FileNotFoundException: 文件未找到。

throw和throws

throw 是用于 抛出异常 的关键字，它后面必须跟一个异常对象（通常是 Exception 类或其子类的实例）。
throw 可以在任何方法体中使用，不受限制。
throw 用来在方法内部抛出异常，并且可以抛出自定义的异常类型。

throws 出现在方法声明中，用于 声明 方法可能抛出的异常。
throws 后面跟的是异常类名，多个异常类型用逗号分隔。
throws 与 检查型异常（checked exceptions）相关，编译器会强制要求处理这些异常。
如果一个方法声明了 throws，调用该方法的代码必须处理这些异常（通过 try-catch 语句或继续抛出）。

多线程

创建线程的三种方式

  1. 继承Thread类

    优点：简单，适合单一任务。
    缺点：Java 不支持多继承，继承 Thread 后无法继承其他类。

  2. 实现Runnable接口

    优点：避免多继承问题，任务与线程分离，代码更灵活。
    适用场景：适用于复杂的多任务系统。

  3. 实现Callable接口

    优点：可以返回结果并处理异常。
    适用场景：需要返回值的任务。

TCP和UDP

  • TCP，传输控制协议(Transmission ControlProtocol)，是面向连接的通信协议。
    使用TCP协议进行数据传输时,两个进程之间会建立一个连接,数据以流的形式顺序传输。

  • UDP，用户数据协议(User DatagramProtocol)，是无连接通信协议。
    使用UDP协议进行数据传输时,两个进程之间不建立特定的连接,不对数据到达的顺序进行检查。

  • 在互联网上进行数据传输,多用TCP和UDP协议，它们传输的都是一个byte stream/字节型的数据流

  • 网络进程就是在网络结点计算机上运行的程序。

  • 网络编程一般是指利用不同层次的通信协议提供的接口实现网络进程通信的编程。

  • Java中网络编程类位于java.net包中

  • URL、URLConnection、Socket和ServerSocket类是基于TCP协议的；

  • DatagramPacket、DatagramSocket和MulticastSocket类是基于UDP协议的。

File

检查文件是否存在、创建文件

读写文本文件

使用 FileReader 和 FileWriter,适合处理文本文件，按字符流读写。

使用 BufferedReader 和 BufferedWriter提供缓冲功能，适合高效处理大文件。

读写二进制文件

使用 FileInputStream 和 FileOutputStream适合读写字节流文件，如图片、音频、视频等。

使用 BufferedInputStream 和 BufferedOutputStream为字节流提供缓冲，提高读写效率。

过滤流

过滤流类的主要功能是为输入/输出流提供一个通用的接口，提供将流连接在一起的能力，即将一个流的输出连接到另一个流的输入，这样可以得到满足应用程序要求的很长的输入和输出过滤器。

  • 缓冲流：借助于字节数组缓存，在读取或存储数据时可以将一个较大数据块读如内存中，或将内存中较大数据块一次性写入指定的文写效率的目的
  • 数据流：主要用来读取于存储基本数据类型的数据。而且数据流的存储格式采用统一的形式，即与具体的计算机无关
  • 对象流
  • 管道流：管道数据流主要用于线程间的通信。
