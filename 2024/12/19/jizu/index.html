时钟同步方法

tsetup

建立时间是指在时钟边沿到来之前，数据输入（D）信号需要保持稳定的最小时间间隔。
如果数据未能在这个时间内稳定，触发器可能无法正确捕获数据，导致计数器或同步逻辑工作异常。

tccq

Clock-to-Q Delay
时钟到输出延迟是指从触发器接收到时钟边沿信号开始，到触发器输出 Q 信号发生变化所需的时间。
它反映了触发器的内部传播延迟。

总线/I/O

总线：
一种共享的通信介质，用于在计算机的各个部件（CPU、内存、I/O 设备）之间传输数据、地址和控制信号。总线按功能分为：

数据总线（Data Bus）：传输数据。
地址总线（Address Bus）：传输内存或 I/O 地址。
控制总线（Control Bus）：传输控制信号（读/写、中断请求等）。

I/O（输入/输出）：
I/O 系统负责在 CPU 和外设之间传输数据。常见 I/O 方式包括：

程序查询 I/O
中断 I/O
DMA（直接内存访问）I/O

总线的仲裁方式

当多个设备需要使用总线时，总线仲裁器（Bus Arbiter）决定谁可以访问总线。常见仲裁方式包括：

  1. 集中式仲裁：
    由一个中心仲裁器负责控制总线的访问。

链式查询法（Daisy Chaining）：
原理：设备按优先级高低串联，总线请求按顺序传递到仲裁器。
优点：简单易实现。
缺点：优先级低的设备可能长期得不到服务。

计时器法（Polling）：
原理：仲裁器以固定顺序轮询设备，询问其是否需要访问总线。
优点：公平，防止设备饿死。
缺点：增加了延迟。

独立请求法（Centralized Parallel Arbitration）：
原理：每个设备有一条独立的请求线，总线仲裁器根据优先级授予访问权限。
优点：高效、灵活。
缺点：需要更多硬件资源。

  1. 分布式仲裁：
    由所有设备共同参与仲裁，不依赖中心仲裁器。

静态优先级法：
原理：每个设备有一个固定优先级，优先级高的设备可抢占总线。
优点：简单。
缺点：低优先级设备可能长期无法使用总线。

动态优先级法：
原理：优先级根据设备请求的历史动态调整。
优点：更公平。
缺点：实现复杂。

总线请求线的个数

总线请求线的个数取决于仲裁方式：

  • 链式查询法：只需一条总线请求线。
  • 独立请求法：每个设备一条请求线。
  • 轮询法：通常只需一条轮询信号线。

程序查询 I/O

定义：
CPU 通过查询 I/O 设备状态寄存器的方式来与设备通信。

工作方式：

CPU 发出 I/O 请求，读取设备的状态寄存器。
查询设备是否就绪（通常通过忙标志位）。
如果设备就绪，CPU 进行数据传输；否则循环等待
优点：
简单易实现。
不需要额外硬件支持

缺点：
CPU 等待设备的过程中无法执行其他任务，效率低。
适合低速设备或简单 I/O。

中断与中断 I/O 方式

中断 I/O 定义：
CPU 不需要持续查询设备状态，而是当设备准备好时主动发送中断信号通知 CPU。

工作方式：
CPU 发出 I/O 请求后继续执行其他任务。
设备完成操作后向 CPU 发出中断信号。
CPU 响应中断，保存当前上下文，调用中断服务程序（ISR）。
ISR 处理数据后，恢复上下文并继续原任务。

优点：
提高 CPU 的利用率，CPU 不需要一直等待设备。
更适合高速或实时设备。

缺点：
中断频繁时可能增加 CPU 开销。
实现复杂，需要硬件中断支持。

DMA（直接内存访问）I/O 方式

定义：
DMA 是一种在设备与内存之间直接传输数据的方式，无需 CPU 的直接参与。

工作方式：
CPU 设置 DMA 控制器（包括数据地址、传输长度等）。
DMA 控制器接管总线，与设备直接交换数据。
数据传输完成后，DMA 控制器向 CPU 发送中断信号。

优点：
极大减轻了 CPU 的负担。
适合大批量数据传输，如磁盘和内存间的数据交换。

缺点：
DMA 控制器需要与 CPU 争夺总线控制权。
增加了硬件复杂性。

方式 特点 优点 缺点
程序查询 I/O CPU 轮询设备状态 简单实现，无需硬件支持 效率低，浪费 CPU 资源
中断 I/O 设备主动通知 CPU CPU 利用率高，适合实时设备 中断频繁可能增加开销
DMA I/O DMA 控制器直接传输数据 高效，减轻 CPU 负担 硬件复杂性高，需争夺总线控制权

链接

  • static 修饰的全局变量具有内部链接性，仅在定义它的文件中可见。

  • 普通全局变量具有外部链接性，可被其他文件访问和修改。

  • 强符号：****函数和已初始化的全局变量，每个强符号在程序中只能定义一次，多个强符号定义会导致链接错误。

    链接器在解析符号冲突时优先选择强符号。

  • 弱符号：****未初始化的全局变量和外部符号

链接的完整流程

  1. 目标文件读取

    链接器加载所有目标文件和库文件。

  2. 符号表构建

    • 扫描所有目标文件，提取符号定义和引用信息，建立全局符号表。

  3. 符号解析

    • 检查符号是否有定义并解决符号冲突。

  4. 地址分配

    • 为目标文件中的代码和数据分配内存地址（通常根据段分配：代码段、数据段等）。

  5. 重定位

    • 调整所有符号的引用地址，使其指向实际的运行时地址。

  6. 生成可执行文件

    • 链接器将所有目标文件和库文件的内容结合起来，生成最终的可执行文件。

目标文件的三种形式

可重定位目标文件（Relocatable Object File）: .o

这种目标文件是编译器的直接输出，尚未链接，主要用于链接阶段。

包含目标代码和数据，以及符号表和重定位表。

符号表记录了已定义符号和未定义符号。

重定位表记录了需要调整的地址信息。

可执行目标文件（Executable Object File） : .exe

共享目标文件（Shared Object File）

段

1 .text 段

  • 功能：存放程序的机器指令，即可执行代码。
  • 属性：通常是只读的，且加载到内存中后标记为不可写。

2 .rodata 段

  • 功能：存储只读数据，例如字符串常量、浮点常量、常量数组等。
  • 属性：只读，防止运行时修改。

3. .data 段

  • 功能：存储已初始化的全局变量和静态变量。
  • 属性：可读可写，在程序加载到内存时初始化。

4. .bss 段

  • 功能：存储未初始化的全局变量和静态变量。
  • 属性：仅占用逻辑地址空间，不占用实际的文件大小。
  • 用途：节省文件空间，因为未初始化变量默认为零，加载时直接分配内存即可。
